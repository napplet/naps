# NAAT-FEED · `feed`

**Recommended open contract:** (none yet)
**Actions:** open

A napplet that displays a scrolling list of many Nostr events selected by some criteria (follows, hashtag, relay, search).

- **IS:** an ongoing, many-event list the user scrolls.
- **IS NOT:** a single event (→ `note`); a real-time stream/video surface.
- **Distinct from:** `note` (many vs. one).

> A NAAT file is this fixed schema and nothing else. Anything that wants to grow —
> payload detail, lifecycle, action-specific wire — belongs in conventions, not here.
